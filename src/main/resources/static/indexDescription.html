Worker API Documentation

1. Получить работника по ID

Endpoint: GET /worker/{id}

Описание: Получить конкретного работника по его идентификатору.

Path Parameter:

  • id (Integer) - Идентификатор работника, которого нужно получить.

Responses:

  • 200 OK - Работник найден.
    • {
    "id": 1
    "name": "string",
    "surName": "string",
    "dateOfBirth": "yyyy-MM-dd",
    "address": "string",
    "create": "yyyy-MM-dd HH:mm:ss",
    "update": "yyyy-MM-dd HH:mm:ss"
}
  • 404 Not Found - Если работника нет.

2. Получить всех рабочих (Limit 5)

Endpoint: GET /worker

Описание: Получить список до 5 последних работников.

Responses:

  • 200 OK - Список объектов Worker (до 5).
    • {
    "id": 0
    "name": "string",
    "surName": "string",
    "dateOfBirth": "yyyy-MM-dd",
    "address": "string",
    "create": "yyyy-MM-dd HH:mm:ss",
    "update": "yyyy-MM-dd HH:mm:ss"
},
        {
    "id": 1
    "name": "string",
    "surName": "string",
    "dateOfBirth": "yyyy-MM-dd",
    "address": "string",
    "create": "yyyy-MM-dd HH:mm:ss",
    "update": "yyyy-MM-dd HH:mm:ss"
}

3. Создать нового работника

Endpoint: POST /worker/new

Описание: Создать нового работника.

Request Body: Объект JSON со следующими полями:

{
    "name": "string (max 50 characters, not blank)",
    "surName": "string (max 50 characters, not blank)",
    "dateOfBirth": "yyyy-MM-dd (must be in the past)",
    "address": "string (not blank)"
}

Responses:

  • 201 Created - Создан объект Worker.
    •   {
    "id": 2
    "name": "string",
    "surName": "string",
    "dateOfBirth": "yyyy-MM-dd",
    "address": "string",
    "create": "yyyy-MM-dd HH:mm:ss",
    "update": "yyyy-MM-dd HH:mm:ss"
    }

4. Обновить существующего работника

Endpoint: PUT /worker/update/{id}

Описание: Обновить существующего работника по идентификатору.

Path Parameter:

  • id (Integer) - Идентификатор работника для обновления.

Request Body: JSON-объект с той же структурой, что и Создать нового работника endpoint.

Responses:

  • 201 Created - Обновлен объект Worker.
    •   {
    "id": 2
    "name": "string",
    "surName": "string",
    "dateOfBirth": "yyyy-MM-dd",
    "address": "string",
    "create": "yyyy-MM-dd HH:mm:ss",
    "update": "yyyy-MM-dd HH:mm:ss"
    }
  • 404 Not Found - Если работника нет.

5. Удалить работника

Endpoint: DELETE /worker/delete/{id}

Описание: Удаление работника по идентификатору.

Path Parameter:

  • id (Integer) - Идентификатор работника, которого нужно удалить.

Responses:

  • 200 OK - Если работник был успешно удален.
  • 404 Not Found - Если работника нет.
